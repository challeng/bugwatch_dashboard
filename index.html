    Bugwatch is a git repository analyzer that gives you real time analytics about code complexity and more.

    Track your (ruby) repositories

    The bugwatch_dashboard project is a Rails application that provides a user interface to visualize analyzed commits. Once you have the server running, it's easy to install a new repository in a few different ways.

    Via Github

    If your project is already on Github, then all you have to do is set up a post-receive hook.

    1. Point the post-receive URL at "/hook" as a relative path to your host. eg: http://example.com/hook
    2. Click "Test Hook" and it will clone and analyze your repository

    Via Git

    If your project is not on Github, then you can install a post-receive GIT hook on the remote repository.

    1. Make sure json gem is installed at a system level
    2. Copy "lib/post_receive.rb" contents into git hooks directory as "post-receive". (GIT_DIR/.git/hooks)
    3. Edit "hook_url" local variable at bottom of post-receive script to point at "/hook" as a relative path to your host. eg: http://example.com/hook
    4. Wait for a new commit to be pushed to fire off the initial payload to begin analyzing

    Create your own analyzer

    Bugwatch can be used as a platform for performing different types of analysis on commits. To be fully compliant, an analyzer must respond to "call" and take a single argument, commit, which is an instance of Bugwatch::Commit.

    class CommitAnalyzer
  def self.call(commit)
    # ... do stuff with commit here
  end
end
